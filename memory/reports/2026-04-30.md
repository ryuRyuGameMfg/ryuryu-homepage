### KPI 改善機序（RSSが有効な理由）

```text
従来の発見経路:
  Google Search → 被リンク経由のクロール → インデックス
  ↓ 被リンクゼロのため詰まる
  KPI = 0

RSS による新発見経路:
  Bing Webmaster RSS Auto-Discovery (24-72h)
  Feedly / Inoreader 購読者経由
  Perplexity / ChatGPT Search の RSS クロール
  ↓ 被リンク不要・robots.txt の Sitemap: 行のみで発見される
  独立した流入チャネル
```

### RSS が「被リンク詰まり」を回避する仕組み

| 仕組み | 説明 |
|--------|------|
| Bing Webmaster Tools RSS取込 | `robots.txt` の `Sitemap:` 行に RSS を含めるとBingが自動検出。Bingのクロールは Google より RSS シグナルを重視 |
| Feedly / Inoreader 表示 | 公開RSSは Feedly Search 等で発見される。AIライターやインフルエンサーがRSS購読する例がある |
| AI 検索エンジン | Perplexity / ChatGPT Search は RSS を優先的にパースし、最新性を担保した回答に引用しやすい |
| robots.txt → Sitemap | Sitemap として登録された RSS は通常のサイトマップと同等にクロール対象に |
